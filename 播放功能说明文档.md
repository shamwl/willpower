# 音乐播放功能说明文档

## 一、播放原理概述

本应用采用**在线流媒体播放**方式，无需下载到本地即可播放音乐。

### 核心流程
```
歌曲URL → AVPlayer → 音频解码 → 播放输出
```

### 技术栈
- **播放引擎**: HarmonyOS AVPlayer API
- **数据来源**: HTTP/HTTPS 在线音频文件
- **协议支持**: 支持 http:// 和 https:// URL

---

## 二、播放流程详解

### 1. 数据来源
歌曲信息存储在 `rawfile/encoded_songs.json` 文件中：
```json
{
  "songs": [
    {
      "id": "001",
      "name": "歌曲名称",
      "url": "https://example.com/music.mp3"
    }
  ]
}
```

### 2. AVPlayer 状态机
```
创建播放器 → 设置URL → 自动进入initialized状态 → 调用prepare() → 进入prepared状态 → 调用play() → 进入playing状态
```

**关键状态说明**：
| 状态 | 说明 | 需要执行的操作 |
|------|------|---------------|
| `idle` | 初始状态 | - |
| `initialized` | URL已设置 | 调用 `prepare()` |
| `prepared` | 准备完成 | 调用 `play()` |
| `playing` | 正在播放 | - |
| `paused` | 已暂停 | 调用 `resume()` |
| `completed` | 播放完成 | 播放下一首或重置 |

### 3. 在线播放流程图
```
┌─────────────────────────────────────────────────────────────┐
│                      开始播放                                 │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 1. 获取歌曲URL (from encoded_songs.json)                      │
│    song.ossPath = "https://xxx.com/song.mp3"                │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. 创建 AVPlayer 实例                                        │
│    const player = await media.createAVPlayer()              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. 设置播放回调                                              │
│    - error 回调: 处理错误                                     │
│    - stateChange 回调: 状态变化通知                          │
│    - timeUpdate 回调: 进度更新                               │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 4. 设置播放源（关键步骤）                                      │
│    player.url = song.ossPath                                 │
│    → AVPlayer 自动进入 "initialized" 状态                   │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 5. 调用 prepare() 准备播放                                   │
│    → AVPlayer 自动进入 "prepared" 状态                       │
│    → 自动触发 stateChange 回调                               │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 6. 状态回调触发: case 'prepared'                             │
│    player.play()                                             │
│    → AVPlayer 自动进入 "playing" 状态                        │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 7. 状态回调触发: case 'playing'                              │
│    更新UI: this.isPlaying = true                            │
│    → 用户听到音乐播放                                        │
└─────────────────────────────────────────────────────────────┘
```

## 三、关键代码说明

### 1. URL 验证
```typescript
if (!song.ossPath.startsWith('http://') && !song.ossPath.startsWith('https://')) {
  throw new Error('无效的URL格式');
}
```
- 必须以 `http://` 或 `https://` 开头
- 本地文件路径不支持（已移除下载到沙箱的逻辑）

### 2. 播放器重置
每次播放新歌曲前，必须释放旧播放器资源：
```typescript
private async resetPlayer(): Promise<void> {
  if (this.audioPlayer) {
    this.audioPlayer.off('error');
    this.audioPlayer.off('stateChange');
    this.audioPlayer.off('timeUpdate');
    await this.audioPlayer.release();
    this.audioPlayer = null;
  }
}
```

### 3. 状态回调处理
```typescript
player.on('stateChange', (state: string) => {
  switch (state) {
    case 'initialized':
      player.prepare();  // 自动调用
      break;
    case 'prepared':
      player.play();     // 自动调用
      break;
    case 'playing':
      this.setPlayState(PlayState.PLAYING);  // 更新UI
      break;
  }
});
```

## 四、HTTP 流媒体播放原理

### 工作机制
```
┌──────────┐    HTTP请求     ┌──────────┐    分块传输    ┌──────────┐
│  App     │ ────────────→  │  服务器   │ ←───────────  │  音频文件 │
│ AVPlayer │                │  OSS/CDN  │              │  (MP3)   │
└──────────┘    HTTP响应     └──────────┘    音频流      └──────────┘
     ↓                              ↓
  接收音频数据                    持续提供
  实时解码播放                    数据流
```

### 优势
1. **无需等待下载**: 边下载边播放
2. **节省存储**: 不占用设备空间
3. **即时播放**: 点击即可播放

### 技术要求
- 网络连接: 需要稳定的网络环境
- 音频格式: 支持 MP3、AAC、FLAC 等常见格式
- CORS 支持: 服务器需要允许跨域访问

## 五、URL 示例

### 有效的在线音频URL
```
https://example-bucket.oss-cn-hangzhou.aliyuncs.com/music/song.mp3
https://cdn.example.com/audio/track.flac
http://your-server.com/files/song.aac
```

### 无效的URL（会导致播放失败）
```
/data/local/song.mp3          ❌ 本地路径不支持
file:///sdcard/music/song.mp3 ❌ file协议不支持
/rawfile/song.mp3             ❌ rawfile路径不支持
```

## 六、常见问题排查

### 1. 播放器状态错误
**错误**: `current state is not stopped or initialized, unsupport prepare operation`
**原因**: 旧播放器未正确释放
**解决**: 确保每次播放前调用 `resetPlayer()` 释放资源

### 2. 网络访问失败
**检查项**:
- 网络权限是否配置（module.json5）
- URL是否可从浏览器访问
- 是否使用 https:// 协议

### 3. 状态回调未触发
**可能原因**:
- 回调注册顺序错误（需在设置URL前注册）
- 旧播放器回调未移除
- 网络问题导致加载超时
